# 新しい開発の開始 (Start Development)

このワークフローは、新しい開発をスムーズに開始するための手順を自動化します。

## 【最重要：開発中の絶対ルール】

### 1. アトミック・コミットの徹底
**「1ファイル更新ごとに、必ず個別にコミットする」**
- `git add [修正ファイル]` を単独で実行する。
- 成功を確認後、`git commit -m "[説明]"` を単独で実行する。
- **注意：`&&` での連結実行は厳禁です。**

### 2. 推測作業の完全禁止 (No Speculation)
**「明示的な指示がない作業、推測に基づく修正は一切行わない」**
- オーナーからの具体的な指示範囲を越えた「先走った修正」は厳禁。
- 指示の意図が曖昧な場合や、必要と思われる修正でも指示に含まれていない場合は、必ず作業前にオーナーに確認し、承認を得ること。

---

## 実行手順

1. **ルールの確認と README の最新化**
   - `docs/guidelines/local_process_rules.md`（AI専用ルール）および `coding_principles.md` を読み込み、最新の規約を確認する。
   - `docs/README.md` のインデックスを実態に合わせて更新する。

2. **現状の理解と指示の待機**
   - `docs/planning/backlog.md` および `docs/dev_refresh_handover.md`（存在する場合）を読み込み、進捗状況を把握する。
   - **勝手な提案は行わず**、現在の状況を整理して報告した上で、オーナーからの具体的な指示を仰ぐ。

3. **開発計画 (dev_implementation_plan.md) の作成**
   - オーナーから具体的な指示を受けたら、詳細な開発計画を **`docs/dev_implementation_plan.md`** として作成する。
   - **モード宣言**: 冒頭に `[SPEC]` または `[IMPL]` を明記する。
   - **連番 (No.)**: 各手順に No. を付与する。
   - **チェックボックス**: `- [ ]` 形式で作成し、進捗を管理する。
   - **ステップ承認**: 各 No. 完了ごとにオーナーの承認を得る旨を明記する。

// turbo
4. [STOP!] 開発計画についてオーナーの承認を得る。
