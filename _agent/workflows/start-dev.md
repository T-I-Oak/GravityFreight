# 新しい開発の開始 (Start Development)

このワークフローは、新しい開発をスムーズに開始するための手順を自動化します。

1. **ルールの確認**
   `docs/rules/` 下の md ファイルをすべて読み込み、コーディング規約やプロセスルールを確認する。

2. **バックログの提案**
   `docs/planning/backlog.md` から「次に着手すべき項目」を選定し、ユーザーに提案する。

3. **開発計画の作成**
   提案が承認されたら、詳細な「開発計画 (Implementation Plan)」を策定して提示する。

// turbo
4. [STOP!] 開発計画についてユーザーの承認を得る。
