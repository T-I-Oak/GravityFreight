# 開発の完了 (Finish Development)

## 【最重要：絶対遵守・安全管理ルール】
- **アトミック・コミット**: 1ファイルごとに個別に git add / commit を実行。
- **Windows対応**: コマンド連結 (`&&`) は厳禁。

---

## 実行手順

1. **仕様書・履歴・バックログの最新化**
   - `docs/specs/` 配下の仕様書を今回の変更に合わせて更新する。
   - **「実装モード [IMPL]」の場合のみ**、`docs/update_history.md` に今回の変更内容を追記する。
   - `docs/planning/backlog.md` から、今回完了したタスクを完全に削除する（アーカイブ不要）。
   - `docs/guidelines/coding_principles.md` への準拠状況をセルフチェックし、報告に含める。

2. **クリーンアップ**
   - `tmp_` ファイルをすべて削除する。
   - `dev_implementation_plan.md` 等の `dev_` ファイルを整理（削除または移動）する。
   - [IMPL] モードの場合、`package.json` のバージョンが最終報告時と一致しているか確認する。

3. **最終承認の獲得**
   - 作業内容の要約をオーナーに提示し、マージの許可を仰ぐ。
   - **オーナーの明示的な承認を得るまで、マージおよびプッシュは行わないこと。**

// turbo
4. [STOP!] マージ・プッシュについてオーナーの承認を得る。

5. **マージとプッシュ**
   - 承認後、フィーチャーブランチを `master` へマージし、リモートへプッシュする。
