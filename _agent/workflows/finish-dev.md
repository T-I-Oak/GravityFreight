# 開発の完了 (Finish Development)

このワークフローは、開発した内容を検証し、プロジェクトドキュメントを整理して master ブランチへ統合する手順を自動化します。

1. **仕様とテストの最新化**
   - `docs/specs/` の仕様書を最新化する。
   - `tests/` 内のテストスクリプトを更新し、`testing_policy.md` に従った構成にする。
   - `npm test` を実行し、全件パスを確認する。

2. **バックログのアーカイブ**
   - `backlog.md` から完了項目を削除し、**`git_workflow.md` の規約通り**に `backlog_archive.md` へ記録する。

3. **[STOP!] 最終承認**
   - 実施した内容を報告し、ユーザーの最終承認を待機する。

4. **統合 (Merge & Push)**
   - 承認後、`master` ブランチへマージし、リモートへプッシュを行う。
