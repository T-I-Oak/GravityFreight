# StartNewDevelopment スキル

このスキルは、プロジェクトで新しい機能開発やバグ修正を開始する際に、一貫した手順（ルール確認、タスク選定、計画策定）を踏むために使用します。

## 【最重要：開発中の絶対ルール】

### 1. アトミック・コミットの徹底
**「1ファイル更新ごとに、必ず個別にコミットする」**
1. `git add [修正ファイル]` を単独で実行する。
2. 成功を確認後、`git commit -m "[説明]"` を単独で実行する。
**注意：`&&` による連結実行は、いかなる場合も厳禁です。**

### 2. 推測作業の完全禁止 (No Speculation)
**「明示的な指示がない作業、推測に基づく修正は一切行わない」**
- 指示範囲を越えた「先走った修正」は厳禁。
- 指示が曖昧、または指示に含まれないが必要と思われる修正がある場合は、必ず作業前にユーザーに確認し、承認を得ること。

## 実行手順

### 1. プロジェクトルールの再確認と README の最新化
以下のディレクトリ・ファイル群を読み込み、現在のプロジェクトにおける規約を最新の状態で把握してください。
- **核心原則 (`docs/V2_CORE_PHILOSOPHY.md`)**: 共通化・階層化・意味論の定義。
- **コーディング原理 (`docs/guidelines/coding_principles.md`)**: 7つの原理と具体的なチェックリスト。
- **UI設計指針 (`docs/specs/ui/design_philosophy.md`)**: 4軸モデル、レイヤー構造、Axis設計。
- **UIコンポーネント基準 (`docs/specs/ui/component_standard.md`)**: セマンティックHTMLと実装パターン。

**ドキュメントの最新化**:
読み込んだ結果、`docs/README.md` のインデックスが実態と乖離している（新しいドキュメントが含まれていない、古いパスが残っている等）場合は、作業開始前に `docs/README.md` を更新してください。

### 2. 現状の理解と引き継ぎの確認
`docs/planning/backlog.md` および関連する仕様書を読み込み、現在の開発フェーズと進捗状況を正確に把握してください。
- **引き継ぎファイルの確認**: もし `docs/local_refresh_handover.md` が存在する場合は、**最優先でその内容を読み込み**、前回のチャットからの継続事項（次に着手すべき No. など）を確認してください。
- **提案は行わない**: AI側から「次はこれをやりましょう」と提案するのではなく、現在の状況を整理して報告した上で、ユーザーからの具体的な作業指示を待ってください。
- **指示の確認**: ユーザーから指示があった場合は、その内容が既存のドキュメントやルールと整合しているかを確認してください。

### 3. 開発計画 (Implementation Plan) の作成
ユーザーから具体的な指示があったら、詳細な開発計画を作成し、必ず**アーティファクト（`.md` ファイル）として保存**してください。計画には以下の項目を含める必要があります。
- **連番 (No.) の付与**: 各手順に必ず No. を振り、意思疎通と進捗管理を容易にすること。
- **目的と要件**: なぜこの変更を行うのか。
- **影響範囲**: 作成・修正するファイルの一覧。
- **具体的な手順**: どのファイルをどのように変更するか、ステップバイステップで詳細に記述する。**（各ステップの最後に、個別にコミットを行う旨を明記すること）**
- **改善提案の提示**: 「改善した方が良い」と思われる独自の提案がある場合は、指示への忠実な実装とは明確に分離し、「提案項目」として詳細（何を・どう・なぜ変えるか）を記述すること。採用の判断はユーザーに仰ぐ。
- **技術的判断**: 実装方針や使用するデータ構造、デザインの方向性。
- **Windows対応の明示**: PowerShell環境であることを考慮し、コマンド結合 (`&&`) を絶対に行わないことの再確認。
- **テスト計画**: どのファイルにどのようなテストコードを書くか。
- **[STOP!] 承認ゲート**: 「開発計画の提示後、ユーザーの承認を得るまで作業を停止する」旨を明記。

## 重要な制約事項 (遵守必須)
- **[最重要] アトミック・コミットの徹底**: 1ファイルごとに git add と git commit を単独で実行。
- **推測の禁止**: 指示されたことのみを行う。
- **コマンド実行**: Windows環境のため `&&` 連結は厳禁。
- **ファイルサイズ**: 1ファイル500行を目安とする。
- **命名規則**: 一時ファイルは `tmp_`、Git管理外は `local_`。
