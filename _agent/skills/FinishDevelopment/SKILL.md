# Skill: finish-dev

## 目的
開発作業を完了し、成果物を整理・反映する。

## 手順
1. ドキュメント類（`docs/` 配下）が最新化されていることを確認する。
    - `backlog.md` 等のタスクリストの整理。
    - バージョン（Patch）がタスク完了ごとに正しく更新されていることを確認する。
    - `update_history.json` 等に、今回の開発内容を Minor 単位（x.y 形式）で追記する。
2. クリーンアップと最終同期を行う。
    - `tmp_` および不要になった `dev_` ファイルの削除。
    - 未コミットの変更がある場合は、アトミック・コミットの原則に従い個別にコミットする。
3. 今回の開発内容のサマリーを報告し、オーナーから完了の承認を得る。
    - 報告には最新のバージョン（Patch を含む x.y.z 形式）を含めること。
4. [STOP!] オーナーの承認を得るまで、マージおよびブランチのクローズを行わないこと。
5. 承認されたら、メインブランチへのマージを実行する。
