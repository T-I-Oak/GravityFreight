# FinishDevelopment スキル

このスキルは、開発作業が完了した際に、プロジェクトの状態を最新化し、品質を担保した上で変更を `master` ブランチへ安全に統合するために使用します。

## 実行手順

### 1. 仕様書の最新化 (Specification Update)
- 今回の開発内容を振り返り、`docs/specs/` 配下の仕様書（`spec.md` 等）に未反映の変更点がある場合は更新してください。
- 実装と仕様の間に乖離がないことを最終確認します。

### 2. テストの最新化と実行 (Testing)
- **テストスクリプトの更新**:
    - `testing_policy.md` に従い、新機能に対応したテストが適切なディレクトリ（`tests/spec/` または `tests/implementation/`）に追加・更新されているか確認してください。
- **全テストのパス確認**:
    - **重要**: Windows環境のため、`npm test` を単独で実行します。
    - すべてのテストが「グリーン（合格）」であることを確認してください。失敗した場合は修正が必要です。

### 3. バックログの整理とアーカイブ (Backlog Archiving)
- **アーカイブへの移行**:
    - `docs/planning/backlog.md` から完了した項目を削除し、`docs/planning/backlog_archive.md` へ移動します。
- **記載ルールの遵守 (git_workflow.md 参照)**:
    - フォーマット: `## Version [Major].[Minor] (YYYY/MM/DD) - [タイトル]`
    - 内容: ユーザー体験に直結する機能的な変更のみを記載し、内部的な作業（テスト追加、リファクタリング、バージョン同期等）はノイズとして除外してください。

### 4. 最終承認の獲得 (Final Approval)
- **[STOP!] ゲートの設置**:
    - ここまでの作業内容（更新した仕様、パスしたテスト結果、アーカイブ内容、マージ予定内容）を要約してユーザーに提示します。
    - **ユーザーの明示的な承認を得るまで、マージおよびプッシュは行わないでください。**

### 5. マージとプッシュ (Merge & Push)
- 承認が得られたら、以下の手順を順番に実行します（Windows環境のため、1コマンドずつ実行）。
    1. `git add .` および `git commit -m "..."` (未コミットがある場合)
    2. `git checkout master`
    3. `git merge [feature-branch]`
    4. `git push origin master`
- マージ完了後、使用したフィーチャーブランチを削除することを検討してください。

## 注意事項
- **Windows対応**: コマンド連結 (`&&`) は絶対に使用しないでください。
- **バージョン同期**: `git_workflow.md` に従い、`Game.js`, `package.json`, `index.html` のバージョンが正しく同期されているか再確認してください。
