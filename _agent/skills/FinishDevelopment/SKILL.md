# FinishDevelopment スキル

このスキルは、開発作業が完了した際に、プロジェクトの状態を最新化し、品質を担保した上で変更を `master` ブランチへ安全に統合するために使用します。

## 【最重要：絶対遵守・安全管理ルール】

### 1. アトミック・コミットの徹底
**「1ファイル更新ごとに、必ず個別にコミットする」**
1. `git add [修正ファイル]` を単独で実行する。
2. 完了を確認後、`git commit -m "[説明]"` を単独で実行する。
**注意：`&&` による連結実行は、絶対に行わないでください。**

### 2. 推測作業の完全禁止 (No Speculation)
**「指示されていない作業をAIの判断で進めない」**
- 指示範囲外のコードの追加やリファクタリング、仕様外の調整を推測で行うことは、システムの不安定化を招くため厳禁。
- 疑問点がある場合は必ず質問し、ユーザーの明確な合意のもとで作業を進めること。

## 実行手順

### 1. 仕様書の最新化と自己レビュー (Spec & Quality Check)
- **仕様の更新**: 今回の開発内容を振り返り、`docs/specs/` 配下の各仕様書に未反映の変更点がある場合は更新してください。
- **[重要] コーディングプリンシプル自己レビュー**: 
    - 今回のブランチで変更した**すべてのソースコード**を対象に、`docs/guidelines/coding_principles.md`（プログラミングの8原則）に準拠しているかセルフチェックを行ってください。
    - 準拠状況を論理的に整理し、ユーザーへの報告内容に含めてください。
- 実装と仕様、および原則の間に乖離がないことを最終確認します。

### 2. バージョン更新と一時ファイル削除 (Pre-test Cleanup)
- **一時ファイルの削除**: `tmp_` で始まる一時ファイルやディレクトリが残っている場合は、すべて削除してください。
- **バージョン同期**:
    - 機能追加や修正がある場合、`package.json` の `version` を更新してください。
    - Vite のビルド環境により、`Game.js` や `index.html` にはビルド・実行時に自動的に反映されます。手動での複数箇所同期は不要になりました。


### 3. 最終承認の獲得 (Final Approval)
- **[STOP!] ゲートの設置**:
    - ここまでの作業内容（更新した仕様、マージ予定内容）を要約してユーザーに提示します。
    - **ユーザーの明示的な承認を得るまで、マージおよびプッシュは行わないでください。**

### 4. マージとプッシュ (Merge & Push)
- 承認が得られたら、以下の手順を順番に実行します（Windows環境のため、1コマンドずつ実行）。
    1. `git add .` および `git commit -m "..."` (未コミットがある場合)
    2. `git checkout master`
    3. `git merge [feature-branch]`
    4. `git push origin master`
    - **自動デプロイ**: `master` ブランチへのプッシュにより、GitHub Actions が起動し、自動的に GitHub Pages (gh-pagesブランチ) へのビルド・展開が行われます。
- マージ完了後、使用したフィーチャーブランチを削除することを検討してください。

## 注意事項
- **[最重要] アトミック・コミットの徹底**: **ファイルを一つ更新・作成するごとに、必ず個別に git add してから git commit を実行すること。**
- **Windows対応**: コマンド連結 (`&&`) は絶対に使用しないでください。
- **キャッシュ対策**: バージョン更新（package.json）は必ずテスト**前**に行うこと。
