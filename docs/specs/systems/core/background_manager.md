# Specification: BackgroundManager Class

## 1. 役割と責務 (Role & Responsibility)

- **所属ドメイン**: System Domain
- **生存期間**: App Lifecycle
- **役割**: 背景描画管理。
- **責務**:
    - 宇宙空間（Starfield）の生成と描画。
    - カメラ状態（CameraController）に応じた背景の同期（回転・視差効果）。
    - セクター遷移時のワープ演出（星の高速移動・光跡描画）の制御。
    - タイトル画面と本編画面で同一の星背景状態を提供し、画面遷移時の連続性を維持する。

## 2. インターフェース (Interface)

### 背景描画仕様

- 本編実画面の背景として、奥行き付きの星空を描画する。
- `WorldRenderer` および `TitleScreenAnimator` から呼び出され、各画面の背面背景レイヤーを担当する。
- `AppOrchestrator` が生成した単一インスタンスを共有し、タイトル画面、セクター開始ワープ、本編マップ表示の間で星配列、奥行き、流速をリセットしない。
- 星は Z 軸の奥行き、明るさ、瞬き周期を持ち、カメラ回転に同期して回転表示される。
- カメラ位置の移動量は 20% の視差として背景に反映され、カメラズームにも追従する。
- ワープ演出中は星を点ではなく光跡として描画し、速度感を表現する。
- セクター遷移時は通常の星空からワープ光跡へ遷移し、到着時に通常の星空へ戻る。

### プロパティ (Properties)
- **`stars: Star[]`**: 
    - 背景を構成する星々のデータ配列。
    - 各 `Star` は `x, y, z`（ワールド相対座標と奥行き）、`size`（大きさ）、`alpha`、`pulseRate`、`pulseOffset` を持つ。
- **`warpSpeed: number`**: 
    - 現在のワープ演出の速度係数。通常時は 1.0。
    - セクター開始ワープ中の最高速度は β v1 と同じ 100.0 とする。
- **`colorPalette: CanvasColorPalette`**:
    - 背景色および星粒子色を `css/design_tokens.css` の token から解決する。

### メソッド (Methods)

- **`initialize(): void`**
    - 初期の星々をランダムに生成・配置する。
    - 視認性を高めるため、遠近感の異なる複数のレイヤー（Z軸の深さ）を持たせる。
    - 初期化済みの星配列がある状態で、タイトル画面と本編画面の切り替えだけを理由に再生成してはならない。

- **`render(context: CanvasRenderingContext2D, view: object): void`**
    - Canvas 2D context と表示領域を受け取り、背景を描画する。
    - `view.width` / `view.height` の範囲を `--color-world-bg` で塗り、生成済みの `stars` を描画する。
    - 星粒子の色は `--color-world-star-particle-rgb` を使用し、フレームごとの明度は alpha として合成する。
    - `warpSpeed > 1` の場合は点ではなく短い光跡として描画する。
    - **最終仕様**: `camera` の状態を参照し、背景を描画する。
    - **投影・同期ロジック**:
        1. **仮想ピボットの算出**: `ScreenCenter - (camera.position * 0.2 * camera.zoomLevel)` を背景の回転中心とする。これにより、パンに対して 20% の視差（遅れてついてくる効果）が発生する。
        2. **星の相対座標の回転**: 各星のデータ `(x, y)` を `camera.rotation` に基づいて回転させる。
        3. **最終位置の決定**: 上記の仮想ピボットを原点として、回転・奥行き投影・カメラズーム適用済みの星の座標を配置する。
    - **ワープ演出**:
        - `warpSpeed` が一定値を超えている場合、星を Z 軸の手前方向へ移動させる。
        - β v1 と同様に、移動前の Z 深度から算出した座標と現在座標を結ぶライン（Streak）を描画し、速度感を演出する。
        - 奥の星など、投影上の Streak が短すぎて線として視認できない場合は点として描画し、ワープ終了時に星が突然増えたように見えない状態を保つ。
        - 手前通過により深度を再配置した星は、画面端から長い線が飛ぶのを避けるため、そのフレームでは Streak を描画しない。
        - ワープ中の星の明度補正は `warpSpeed` に追従して補間し、減速中に通常明度へ滑らかに戻す。

- **`update(deltaSeconds: number): void`**
    - 前回描画からの経過秒数を受け取り、星の奥行き位置とワープ速度を更新する。
    - 通常時も星はゆっくり手前方向へ移動し、画面手前を通過した星は奥の深度へ再配置される。
    - ワープ速度の変更中は、設定された遷移時間に従って `warpSpeed` を補間する。

- **`startWarpEffect(duration: number): void`**
    - 航行開始やワープ突入時に、背景の星々を最高速まで加速させて「超高速移動感」を演出する。
    - 最高速度は 100.0 とし、β v1 のセクター開始ワープと同じ速度感を基準にする。
    - `duration` はミリ秒で指定する。省略時または 0 以下の場合は即時に最高速へ切り替える。
    - 速度の上昇に伴い、星の点が長い光跡（Streak）へと変化し、画面全体に躍動感を与える。

- **`stopWarpEffect(duration: number): void`**
    - ワープ出口に接近した際、最高速から通常速度（1.0）まで減速させ、新セクターへの「到着感」を演出する。
    - `duration` はミリ秒で指定する。省略時または 0 以下の場合は即時に通常速度へ切り替える。
    - 光跡が徐々に短くなって元の点へと戻り、静かな星空へと遷移する。

- **`handleResize(width: number, height: number): void`**
    - 画面サイズ変更に合わせて、描画領域や星の配置密度を調整する。
