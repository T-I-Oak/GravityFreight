# Achievement System Specification (アチーブメントシステム仕様書)

## 1. 概要 (Overview)
プレイヤーの活動を統計（Stats）として記録し、特定の条件を満たした際に「称号（アチーブメント）」を付与するシステム。トースト通知は日本語を採用し、瞬間的な理解を促す。

---

## 4. ストーリー実績 (Story Achievements)

ストーリーは施設（T / R / B）への累積配達によって進行する 3段階分岐（計39話）の構造を持つ。

### 4.1 読了数実績
| ID | 表示説明 | Tier I | Tier II | Tier III (COMPLETE) |
| :--- | :--- | :--- | :--- | :--- |
| stat_stories_read | 既読ストーリー数 | 聞き手 (5) | 語り部の守人 (20) | 物語の全知者 (39) |

### 4.2 コンプリート実績 (特例報酬)
| ID | 表示説明 | Tier MAX (COMPLETE) | 達成条件 |
| :--- | :--- | :--- | :--- |
| stat_t_branch | ストーリー「だれかの願い」 | 交易所の語り部 (13) | Tから開始するストーリーを13個すべて既読にする |
| stat_r_branch | ストーリー「つくる人たち」 | 整備工場の語り部 (13) | Rから開始するストーリーを13個すべて既読にする |
| stat_b_branch | ストーリー「せかいの仕組み」 | 闇市場の語り部 (13) | Bから開始するストーリーを13個すべて既読にする |

※ 特例のストーリー実績も、内部的には他の実績と同様の「段階進行（最大13）」を持つ標準的な実績として算出・レンダリングされる。

## 5. デザイン方針 (Visual Style)

### 5.1 ポップアップ通知 (Achievement Toast)
- スタイル: 施設画面やゲームオーバー画面と親和性の高い、マットデザイン（未来のチラシ風）。
- 非発光: マップ画面のネオン効果と混ざらないよう、Glow（発光）効果を排したソリッドな背景色を使用。
- 視認性: 瞬間的な理解を促すため、大きな日本語フォントと簡潔な情報（称号・数値）を主体とする。

### 5.2 UI全般
- カラーパレット: 青白系のネオンパレットではなく、白・黒・グレー、またはトーンを抑えたアクセントカラーを使用。
- 絵文字: 世界観維持のため一切の使用を禁止。アイコンは既存のシンボルを使用。

## 6. 技術仕様 (Technical Specifications)

### 6.1 データ構造
アチーブメントシステム自体は game_stats キーに集計データを保持するが、ストーリー関連の実績判定については、既存の StorySystem が管理する以下のデータを直接参照する。

実績データ本体 (gravity_freight_game_stats)
```json
{
  "stats": {
    "stat_runs": 102,
    "stat_launches": 1012
  },
  "unlockedIds": ["stat_runs_1", "stat_runs_2"]
}
```

ストーリー進捗データ (既存: gravity_freight_story_progress)
※ StorySystem が管理。アチーブメント側はこのデータの内容を判定に使用する。
```json
{
  "readIds": ["T", "TR", "BBB", "R", "RT", "RTT"]
}
```
- ID法則: T (エピソード1), TR (エピソード2), TRB (エピソード3) といったブランチ履歴文字列。
- 全39種類: 全ストーリー制覇は、stat_stories_read の Tier III にて達成。

### 6.2 通知システム
- 実績解除時に AchievementToast クラス経由で画面上部に通知を表示。
- 同時に複数が解除された場合はキューイングされる。

### 6.3 UI表示
- タイトル画面の「ACHIEVEMENTS」ボタン、またはターミナルの「ARCHIVE」ボタンからアクセス可能。
- **階級印 (Tier Seal)**: 達成段階に応じて「TIER 1」〜「TIER 7」のスタンプが付与される。ローマ数字またはアラビア数字を使用し、書体にはセリフ体（Georgia等）を使用。
- **カラーパレット**: 印刷スタイル（ui-style--printing）のスタンプカラーを基準とした 7段階のティアカラーが適用される。
    - **Tier 1 (最高位)**: `stamp-ss` カラー
    - **Tier 2**: `stamp-s` カラー
    - **Tier 3**: `stamp-a` カラー
    - **Tier 4〜7**: `stamp-b`〜`e` カラーに順次対応
- **ロック状態**: 未解除の実績はグレースケールおよび透過表示（opacity: 0.5）され、称号部分に「NOT ACHIEVED」のラベルが表示される。
- **進捗表示**: 次の Tier までの進捗をゲージ（log-gauge）および数値（current / goal）で表示する。最高 Tier 達成後は「MAX」と表記される。
