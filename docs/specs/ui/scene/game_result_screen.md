# UI仕様: ゲームリザルト画面 (Terminal Report)

## 目的
- ゲーム終了時に、最終結果（スコア、グレード、ランキング等）を「公的な報告書」の形式で提示する。

## 外観・演出 (Visuals & Animation)
- **スタイル**: 印刷スタイル (`ui-style--print`) を採用。
- **物理表現**:
    - **排出アニメーション**: プリンターから段階的に紙が吐き出されるメカニカルな動き (`receiptPrintFeed`)。
    - **衝撃表現**: スタンプが押される瞬間に、レポート全体が物理的な衝撃で揺れる演出。
- **テクスチャ**: ギザギザの切り取り端、インクのにじみ、ドットマトリクス風のタイポグラフィを再現。

## 主要コンポーネント
1.  **ヘッダー**: レポートタイトルと会社名。
2.  **統計セクション**:
    *   到達セクター数 (`num-sector`)
    *   コレクション数 (`num-item-count`)
3.  **最終評価セクション**:
    *   最終スコア (`num-score`)
    *   ランキング表示: 1位〜3位の場合は数値部分を赤インクで強調。
4.  **スタンプゾーン**:
    *   最終グレードに応じた「公印」が押される。
    *   スタンプは着地時にランダムな回転（±30°）と位置ズレ（±20px）が発生し、アナログな質感を出す。
5.  **バーコード**: 事務的な詳細度を高めるための装飾。
6.  **フッター**: タイムスタンプと「契約終了」ボタン。

## 実装構造 (3層アーキテクチャ)
- **Base 層 (`ui_base.css`)**: `.ui-overlay`, `.ui-panel`, `.ui-row` 等の共通レイアウト骨組み。
- **Style 層 (`ui_style_print.css`)**: 紙の地色、インク色、フォント、発光禁止設定、スクロールバー。
- **Screen 層 (`game_result.css`)**: 排出アニメーション、スタンプの配置、バーコード、衝撃揺れ等の画面固有ロジック。

## レイアウト安定性
- **Layout Confinement**: スタンプ出現時の巨大なサイズが紙全体のレイアウトに干渉しないよう、スタンプゾーンには `contain: size layout` を適用。
- **Scrollbar Stability**: `scrollbar-gutter: stable` により、コンテンツの長さに関わらず横揺れのない安定した中央配置を維持する。

## 今後定義が必要な振る舞い (Pending Behaviors)
以下のロジックおよび機能定義は、本仕様書には含まれておらず、今後の開発フェーズで確定させる必要がある。
- **評価（GRADE）算出ロジック**: スコア、セクター数、コレクション数から最終グレードを算出する具体的な計算式。
- **ランキング反映条件**: どのタイミングでローカルストレージのランキングに保存し、上位判定を行うかのフロー。
- **リプレイ連携**: リザルト画面から「ベストショットのリプレイ」を再生・共有する機能の有無と導線。
- **シーン遷移の確定条件**: 「END CONTRACT」ボタン押下後の、フェードアウト時間や初期化プロセスの詳細定義。
- **多言語対応**: 数値単位（pts, scs等）や、スタンプ内のテキスト（OPERATOR AUTH. 等）のローカライズ方針。
