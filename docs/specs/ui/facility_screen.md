# UI仕様: 施設画面

## 目的
- セクター成功後に発生する拠点イベント（交易所/整備工場/闇市場）を提供する。

## 想定する状態（シーン）
- FACILITY

## 施設種別
- TRADING_POST
- REPAIR_DOCK
- BLACK_MARKET

## 主要UI
- 施設名/説明
- 所持金/所持品
- 取引/整備/購入の操作UI
- 次セクターへ進む導線

## 入力
- マウス/タップ操作

## 遷移
- FACILITY → PLAY（次セクター開始 / BUILDING）
- FACILITY → GAME_RESULT（次セクター開始条件を満たせずゲームオーバーとなる場合）

## 表示スタイル
- **原則**: マットスタイル（`ui-style--matte`）
- **構成**: 
    - 2カラム等幅（50% / 50%）のレイアウトを標準とする。
    - 各カラム内は `.facility-section` によって構造化し、見出しと内容の距離を適切に（密に）保つこと。
- **カプセル化**: 
    - 施設固有のスタイル（`facility.css`）は、必ず `#facility-screen` ID を使用してカプセル化し、グローバルへのスタイルの漏洩を防止すること。
- **色彩管理**: 
    - 施設固有のイメージカラーは、JSまたはCSSクラス（`.is-trading-post`等）によって `--facility-color` 変数へ割り当てる。直接の色コード指定は禁止し、`.facility-name` やアイコンの枠線にこの変数を参照させる。
