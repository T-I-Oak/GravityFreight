# UI Spec: Analytic Archive (記録画面) - V2

## 1. 目的と役割
プレイヤーのこれまでの航行記録、実績（Achievements）、およびリプレイデータを閲覧・分析するための統合ターミナル。宇宙船のメインシステムの一部として、高い情報密度と実用性を備えたデザインとする。

## 2. デザインコンセプト
- **スタイル**: 原則として「Matte（マット）」スタイルを適用。
- **レイアウト**: 1120px 幅のワイドな 2 カラム構成（サイドバー + メインエリア）。
- **セマンティクス**: 
    - 統計情報には `.ui-stat-label` を使用。
    - 凡例には `.is-legend` 修飾子を付与し、カテゴリー色を反映したカラーラインを疑似要素（`::before`）で表示。

## 3. UI 構成要素

### 3.1 共通ヘッダー
- **タイトル**: 「Analytic Archive」およびサブタイトルを表示。`.ui-split-column` を使用。
- **タブ切り替え**: Analytics, Achievements, Replays の 3 つの主要カテゴリを切り替える `.ui-tab-group` を配置。

### 3.2 サイドバー (Left Pane)
- 累積統計（KPI）を一覧表示。
- **Lifetime Contracts**, **Cumulative Sectors**, **Total Items** などの重要指標を `.ui-well` 内に表示。
- 幅は 240px 固定（PC時）。

### 3.3 メインエリア (Right Pane / Main Content)
- **Analytics タブ**:
    - **パフォーマンス推移グラフ**: 最近のセッションのスコア・セクター・回収アイテムの推移を視覚化。右上に `.is-legend` による凡例を表示。
    - **ベストランキングテーブル**: 過去の最高記録をランキング形式で表示。`.ui-well` と `.ui-scrollable` を組み合わせたスクロール可能なテーブル構造。

## 4. レスポンシブ設計
- **ブレークポイント**: 800px 以下。
- **レイアウト変更**: サイドバーの固定幅を解除し、1カラムの縦積み構成に変更。
- **一貫性の維持**: モバイル表示時のパネルサイズをプロジェクト標準の **`95vw` / `90vh`** に調整し、周囲に適切な余白を確保する。
- **グラフの保護**: モバイル時でもグラフが突き抜けないよう、`.ui-well` と `.graph-svg` に対して `overflow: hidden` および `min-width: 0` の制約を適用する。

## 5. 実装構造
- **Base 層**: `ui_base.css`, `ui_layout.css`, `ui_primitives.css` を使用。
- **Style 層**: `ui_style_matte.css` による質感を適用。
- **Context 層**: `css/archive.css` で画面固有のパネルサイズ、カラム比率、凡例装飾を定義。
