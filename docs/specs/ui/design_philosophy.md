# UI Design Philosophy: Gravity Freight

## 1. 目的
この文書は Gravity Freight の UI が守るべき表現方針と CSS 設計方針を定義する。
クラス命名、保守性、共通スタイルの責務は GameWorks OAK 共通仕様を上位ルールとし、本書は Gravity Freight 固有の判断基準だけを補足する。

## 2. 上位ルール
UI/CSS の実装および仕様化では、GameWorks OAK の共通 Style Guide に従う。

- `!important` は原則禁止する。
- inline style は避ける。ただし、座標、進捗率、アニメーション値など JavaScript で動的計算する値に限り例外として許可する。
- クラスは役割に応じて次の階層で定義する。
    - `.theme-*`: アプリ全体、画面群、または大きな領域のテーマ。
    - `UpperCamelCase`: 機能、構造、レイアウトを担う共通部品。
    - `.texture-*`: 表面的な質感、色、影、光沢など。
    - `.state-*`: 選択、無効、強調、通知などの動的状態。
    - `lower-case`: その要素が何であるかを示す種別名。
- 機能と質感を分離し、部品の構造を変えずにテーマや質感を差し替えられるようにする。

## 3. Gravity Freight の UI アイデンティティ
Gravity Freight の UI は、単なるメニューではなく、宇宙船やステーションに存在する業務端末として扱う。

- 画面はゲーム世界内の操作盤、記録端末、航行支援装置として見えることを優先する。
- 派手な装飾よりも、情報の読みやすさ、操作の迷いにくさ、繰り返し使う画面としての落ち着きを優先する。
- 色、数値、状態表示は意味を持つ情報として扱い、装飾だけを目的に増やさない。
- ネオンや発光表現を使う場合も、重要状態、航行情報、警告などの意味を持つ箇所に限定する。

## 4. 没入感と操作性
没入感は、ブラウザ標準 UI を隠すこと自体ではなく、ゲーム内の操作として自然に理解できることを目的とする。

- `alert()` や `confirm()` などのブラウザ標準ダイアログは使わず、ゲーム内 UI として表現する。
- ヘルプや補足説明は、HTML の `title` 属性に依存せず、必要に応じてゲーム内 UI として提供する。
- キーボード操作、フォーカス、スクロールは、操作不能や確認困難にならない範囲でゲームの見た目に合わせる。
- フォーカス状態を完全に消すのではなく、ゲームの表現に合う `state-*` または専用表現として定義する。
- スクロールバーを装飾する場合も、領域がスクロール可能であることが分かる状態を維持する。

## 5. 数値と情報表現
数値や状態は、ゲーム内で同じ意味を持つものが同じ形式で表示されるようにする。

- 数値は原則として 3 桁カンマ区切りで表示する。
- 単位は仕様で定義された意味に従い、画面ごとに独自表記を増やさない。
- スコア、コイン、セクター、耐久度、ランク、実績などは、見た目ではなく意味に基づくトークンまたは種別で扱う。
- 色だけで意味を伝えず、ラベル、アイコン、配置、状態名などと組み合わせて判別できるようにする。

## 6. CSS 設計方針
Gravity Freight の CSS は、共通スタイルと画面固有スタイルを分けて管理する。

- 共通スタイルは、複数画面で再利用される構造、部品、状態、質感を定義する。
- 画面固有スタイルは、特定画面の配置、サイズ調整、画面専用アニメーションに限定する。
- 画面固有スタイルは、その画面のトップレベル要素を起点にスコープを閉じる。
- 共通スタイルの用途と境界は `common_style.md` に定義する。
- 個別コンポーネントの構造と表示データは `component_standard.md` に定義する。

## 7. 旧仕様からの移行
過去の文書や実装に共通 Style Guide と一致しない命名が残っている場合でも、新規仕様と新規実装では共通 Style Guide の命名階層を優先する。

既存の詳細仕様を更新する場合は、単純な置換ではなく、クラスの責務を確認して次のいずれかへ分類する。

- 構造または機能を担うもの: `UpperCamelCase`
- 質感を担うもの: `.texture-*`
- 状態を担うもの: `.state-*`
- 要素の種別を表すもの: `lower-case`
- 画面全体の文脈を担うもの: `.theme-*` または画面固有の lower-case 種別
