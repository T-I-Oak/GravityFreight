# Gravity Freight V2: Development Backlog

この Backlog は未完了タスクのみを管理する。
作業主体は以下の区分で明示する。

- **Codex 作業**: 開発、修正、仕様更新、テスト追加、自己レビュー。
- **Owner 確認**: 実ゲームまたはブラウザでの確認。Owner が確認完了と明言したら削除する。
    - Owner 確認項目の ID は必須ではない。必要に応じて、完了確認時の意識疎通を容易にするため `OC-*` の ID を付与する。
    - 複数画面、複数施設、複数操作にまたがる確認は、完了判定できる単位へ分解する。
    - 指摘修正後は、修正確認依頼として新しい Owner 確認項目を追加する。
- **完了条件不明**: そのままでは完了判定できないタスク。削除するか、完了条件を定義して移動する。

## Codex 作業

### 作業ルール
- 不明点や仕様判断が必要な点は、推測で進めず Owner に確認してから対応する。
    - 既に仕様が明確なものは実装・テスト・仕様更新を進める。
    - 仕様が不明なもの、複数の解釈があり得るもの、ゲームルールに影響するものは、勝手に補完しない。
    - バグの隠蔽や過剰な防衛処理になり得る対応は、原因と仕様を確認してから実装する。
    - バグを隠す fallback は明確なルール違反として扱い、欠落してはならないデータや DOM は例外で検出する。
