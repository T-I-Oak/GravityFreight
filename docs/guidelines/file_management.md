# File Management Guidelines: Gravity Freight V2

本ドキュメントは、プロジェクトにおける非公開（Git管理外）ファイルの命名規則と運用ルールを定義する。

## 1. 概要 (Overview)
Gitリポジトリに含めないファイル（`.gitignore` で除外されるファイル）は、その役割と生存期間に応じて以下の3つの接頭辞を使い分ける。

## 2. 分類と接頭辞 (Categories and Prefixes)

### 2.1 tmp_ (Temporary)
*   **分類**: 一時ファイル
*   **生存期間**: 数分〜数時間（タスク実行中のみ）
*   **用途**:
    *   開発中の断片的なコードの動作確認。
    *   一時的なデバッグ出力。
    *   特定の機能を一度だけ検証するための使い捨てスクリプト。
*   **運用ルール**:
    *   **開発完了（タスク終了）までに必ず削除**すること。

### 2.2 dev_ (Development-period)
*   **分類**: 開発期間内維持ファイル
*   **生存期間**: 数日〜数週間（ブランチ作成からマージまで）
*   **用途**:
    *   AI（Antigravity）とユーザー間の引き継ぎ資料（Handover）。
    *   詳細な開発計画（Implementation Plan）。
    *   開発期間中に参照が必要な一時的な仕様メモや、中長期的な検証用モックアップ。
*   **運用ルール**:
    *   **ブランチのマージ（開発フェーズの終了）時に整理・削除**すること。

### 2.3 local_ (Local-only)
*   **分類**: ローカル保存ファイル（オーナー管理）
*   **生存期間**: 永続（オーナーが必要とする限り）
*   **用途**:
    *   リポジトリには含めないが、その開発環境において永続的に維持すべき資産。
    *   オーナー専用の運用ルール、ツール、検証環境設定など。
*   **運用ルール**:
    *   **オーナー管理**の領域であり、オーナーの明示的な指示がない限り、これらを変更・削除してはならない。

## 3. 設定 (Configuration)
これらのファイルが誤ってGitにコミットされないよう、`.gitignore` には以下の設定が必須となる。

```ignore
tmp_*
dev_*
local_*
```
