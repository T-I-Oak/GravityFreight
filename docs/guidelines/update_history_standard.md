# Update History Standard: Gravity Freight V2

## 1. 目的
`update_history.md` は、プロジェクトの進化を「ユーザー体験（機能・品質）」の観点から記録するためのものである。

## 2. 記述の基本原則
- **ユーザー体験優先**: プレイヤーや運用者に直接影響のある「追加機能」「UI改善」「不具合修正」のみを記載する。
- **内部作業の排除**: リファクタリング、ファイル分割、テスト作成、ライブラリ導入などの「内部作業ログ」は記載しない。
- **簡潔な記述**: 専門用語を避け、変更によって何が可能になったか、何が改善されたかを簡潔に記す。

## 3. 記述フォーマット
## Version [X.Y.Z] (YYYY/MM/DD) - [タイトル：アップデートの主要テーマ]

### 機能追加 (Feature)
- 新しく追加された機能やコンテンツ。

### 機能改善 (Enhancement)
- 既存機能のブラッシュアップ、UI/UXの向上、パフォーマンス改善、データ整合性の強化など。

### 不具合修正 (Bugfix)
- ユーザーに影響のあった挙動の修正。
