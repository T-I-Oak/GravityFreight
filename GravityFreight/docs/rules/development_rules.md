# Gravity Freight 開発運用ルール

本プロジェクトを健全に維持・開発するためのエンジニア向けルールです。

## 1. バージョン管理

### 1.1 バージョン表記
アプリのバージョンは、セマンティックバージョニング（Major.Minor.Patch）に準拠して管理します。
- 現在のバージョンは `GravityFreight/src/Game.js` の `this.version` に定義されています。
- バージョン情報は、ゲーム画面の左下（デバッグ・情報レイヤー）に表示されます。


### 1.2 バージョン更新ルール
アプリのバージョンは、セマンティックバージョニング（Major.Minor.Patch）に準拠して管理します。
- **Major (メジャー)**: ユーザーから明示的な指示があった場合のみ上げます。
- **Minor (マイナー)**: 今回の開発（ブランチ）ごとに1つ上げます。
- **Patch (パッチ)**: 実装を修正・微調整するごとに1つ上げます。


## 2. キャッシュ無効化対応

ブラウザキャッシュによる古いスクリプトの実行を防ぐため、`index.html` での JS 読み込み時にクエリパラメータを付与します。

### 2.1 実装方法
`index.html` 内の `<script>` タグの末尾に `?v=X.X.X` の形式でバージョンを指定します。

```html
<script src="GravityFreight/src/main.js?v=0.1.0"></script>
```

### 2.2 更新手順
バージョン（`Game.js` の `this.version`）を更新した際は、**必ず `index.html` 内の全てのスクリプトタグに付与されている `?v=X.X.X` の値も同じバージョンに書き換えてください。**

## 3. 著作権表記
- ゲーム画面左下に適切な著作権および制作年を表示します。
- 描画ロジックは `GravityFreight/src/Renderer.js` の `drawVersion` メソッドで管理します。
- 表記形式: `© T.I.OAK 2026`

## 4. コーディング原則

### 4.1 「念のため」のガードの禁止
実装の不備を隠蔽し、バグを顕在化しにくくするような防御的コード（「念のため」のプロパティ存在チェックや、不正な状態を暗黙的にスキップする処理など）は**原則禁止**します。
- **理由**: バグを隠すことで修正が遅れ、将来的なバグの温床となるためです。
- **方針**: 不正な値や予期せぬオブジェクトが渡された場合は、例外を投げる、あるいはシミュレーションを停止させるなど、不具合が即座にフィードバックされる状態を保ってください。
- **正当な補完**: クラスに本来あるべきメソッドの不足や、仕様に基づいた境界条件の処理を追加することは、ガードではなく「正当な実装」とみなされます。

## 5. CSS / スタイリング

### 5.1 !important の使用禁止（原則）
CSS において `!important` を使用することは**原則禁止**します。
- **理由**: 詳細度の管理が困難になり、将来的なスタイルの上書きやメンテナンスを著しく阻害するためです。
- **方針**: 他のスタイルを上書きする必要がある場合は、セレクタの詳細度を高める（`.class1.class2` のように連結する）か、定義の順序を見直すことで対応してください。
