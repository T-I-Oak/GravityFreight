# プロジェクト基本指針 (Project Standards)

本ドキュメントは、Gravity Freight プロジェクトにおける開発の基本原則を定めます。開発者は、本指針を最優先で遵守してください。

## 1. ファイル・フォルダ管理
- **ファイル命名**:
    - **local_**: リポジトリには含めないが、この環境における永続的な管理のために使用する（例: `local_settings.json`）。
    - **tmp_**: 一時的に作成するファイルで、いつ消しても問題がないもの（例: `tmp_debug.js`）。作業完了時に削除することが推奨される。
- **Git管理**:
    - `tmp_` および `local_` で始まるファイル/フォルダは `.gitignore` で管理対象外とする。

## 2. 著作権表記
- ゲーム画面左下に適切な著作権および制作年を表示します。
- 描画ロジックは `GravityFreight/src/Renderer.js` の `drawVersion` メソッドで管理します。
- 表記形式: `© T.I.OAK 2026`

## 3. ドキュメント優先の原則
- **仕様の最新化**: `GravityFreight/docs/specs` 以下の仕様書は常に最新の実装を反映させること。
- **乖離の防止**: 実装前に仕様を確認し、実装が仕様から乖離する場合は、まず仕様を更新してから実装に着手すること。

## 4. 実装の粒度とリファクタリング
- **ファイルサイズ**: 1ファイルあたり 500 ステップ程度を上限の目安とする。
- **リファクタリング**: 上限を超えた場合、または密結合が目立つ場合は、即座にリファクタリング（機能の分離・抽出）を提案し、実行すること。
