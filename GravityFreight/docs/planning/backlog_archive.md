## Version 0.7 (2026/04/02) - 仕様整合性と経済システムの安定化

### 仕様整合性の確保
- [x] **目的地報酬および配送ボーナスの数値検証とロジック修正**: 基礎報酬 (+2000~5000) とボーナス (+1500) の合算を検証・修正。
- [x] **出現率とセクターしきい値の重み付け抽選の検証**: `14 + S` の計算式とレアリティごとの重み付けが仕様通りであることを検証。
- [x] **複数保険モジュールの重複支払い検証**: 保険モジュールの装備数に応じた保険金支払いロジックの正当性を検証。

### 貨物・経済・バランス調整 (Economy & Balance)
- [x] **デバッグコードの削除と配置数の正常化**: `MissionSystem.js` 内のアイテム強制注入コードを削除し、星ごとのアイテム配置数（1〜2個）の制約を遵守。
- [x] **ゴール通過報酬の正常化**: 貨物を所持していない場合でも、ゴール到達時にボーナスアイテムが付与されるように改善。
- [x] **アイテムデータの日本語化と仕様書整合**: `Data.js` の貨物アイテム名を日本語に統一し、`spec.md` の説明文と完全一致するように修正。

## Version 0.10 (2026/04/04) - モバイル対応、UI 刷新、および不具合修正

### 入力・操作性 (Controls & Mobile)
- [x] **iPad / Android 対応**: `pointer/touch` 経由での入力対応。2本指ドラッグ（パン）、2本指ピンチ（ズーム）の実装。
- [x] **入力精度の向上**: 
    - ズーム判定へのデッドゾーン導入。
    - パニング後の回転ピボット座標補正（回転方向の不自然さ解消）。
- [x] **マップ操作の改善**: パニング後のマップ回転挙動の修正。ブランチ `feature/fix-input-precision` による適正化。

### 物理挙動とゲームシステム (Physics & Game System)
- [x] **予測線と実軌道の同期**: `aiming` 開始時にロケットの質量を `ship.mass` へ同期し、シミュレーション精度を保証。
- [x] **アイテム配置と回収の正常化**: 
    - 母星上のアイテムも回収対象に含まれるよう修正。
    - アイテム配置を仕様に基づく 1〜2個のランダム配置に復元。
- [x] **配送ボーナスの修正**: 配送成功時の表示名を `Cargo Bonus` に統一。

### UI・演出の改善 (UX/UI)
- [x] **UI デザインの刷新**: 
    - 「ネオン（タイトル等）」と「マット（施設等）」の二極化デザインを確立。
    - `color-mix` によるカラーテーマ同期システムの導入。
- [x] **タイトル・レスポンシブ対応**: ロゴの自動折り返し、ワイドメニューボタン、著作権表記の調整。
- [x] **リザルト画面の最適化**: 
    - ステータスバッジ (`DELIVERED / UNMATCHED`) の共通化。
    - 獲得アイテムリストのレイアウト崩れ解消。
    - 専用スクロールバースタイルの統一。
- [x] **アクセシビリティ**: 
    - 星のホバー情報の表示不具合修正（PointerMove 追跡修正）。
    - 情報パネルのはみ出し防止、`z-index` 調整。
- [x] **バグ修正**: 
    - 同一 ID アイテム（シャーシ等）が同時に選択状態になる不具合の修正（`instanceId` による識別徹底）。
    - リザルト画面のアイテムバッジ重なり解消。

## Version 0.11 (2026/04/04) - 貨物配送報酬の適正化とアイテム管理の精緻化

### 経済システムと報酬ロジック (Mission & Economy)
- [x] **cargo 配送報酬の N 倍化**: 
    - 一致する貨物の個数 $N$ に応じて、基本報酬と配送ボーナスを累積計上するように修正。
    - ゴール到達自体の基本報酬（Spec 6.1）は、貨物の有無に関わらず 1 回分を常に保証。
- [x] **アイテムガチャの厳格化**: ボーナスアイテム（Gacha）の付与を「一致配送時のみ」に限定し、 cargo なし/不一致時の余剰付与を修正。
- [x] **リザルト内訳の分離**: `Goal Reward`（基本）と `Delivery Bonus`（ボーナス）を個別の項目としてリザルト画面に表示。

### アイテム・耐久力管理 (Items & Durability)
- [x] **発射台の耐久力消費バグの修正**: 
    - スタックされたアイテム（発射台等）を使用する際、スタック全体ではなく「1個のみ」を消費し、残りをインベントリに正しく再マージするフローを確立。
    - 使用後のアイテムと既存スタックの自動結合ロジックを `InventorySystem.js` に実装。
- [x] **累積アイテム取得数の精度向上**: `addItem` に `isNew` フラグを導入。再マージや差し戻し時に、新規取得として二重カウントされない仕組みを導入。
- [x] **発射台の名称・性能調整**: モデル名 (`LN-1500/2`, `LN-1800/d4`) と価格体系を仕様に沿って最適化。

## Version 0.12 (2026/04/04) - UI 調整とスタック表示のプレミアム化

### UI・演出の改善 (UX/UI)
- [x] **リザルト画面のボタン配置修正**: `VIEW MAP` や `CONTINUE` ボタンが左寄せになっていた問題を修正し、中央揃えに復元。
- [x] **アイテムカードのスタック数・耐久力表示の統合**: 
    - 従来排他的だった「個数表示 [x N]」と「耐久力ゲージ」を同時に表示できるように `UIComponents.js` のロジックを刷新。
    - 個数が 1 の場合はバッジを非表示にし、情報密度を最適化。
- [x] **プレミアム・スタックバッジの導入**: 
    - 単なるテキスト表示から、カテゴリーカラーを基調とした反転カラーデザイン（明るい背景に濃い文字）へアップグレード。
    - CSS `clip-path` を用いた「重なったカード」アイコンをバッジ内に実装。
    - ネオングロー効果と `box-shadow` による視認性の向上。
- [x] **ロケット内モジュール表示の最適化**: 
    - ロケット詳細パネル内のモジュールリストでも、スタック数とゲージを統合表示。
    - 狭いスペースに対応した `.mini` バリアントを実装し、フォントサイズとアイコンサイズを微調整。
