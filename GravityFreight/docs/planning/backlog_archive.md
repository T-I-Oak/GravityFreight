## Version 0.20 (2026/04/06) - レアアイテムの機能復元と物理挙動の刷新

### ゲームシステム (Game System)
- [x] **ゴーストモジュールの物理挙動刷新**: クッション・ゴースト等の予測線と実際の航行が一致しない不具合を修正。バウンド時の「重力粘着」を防止するため、回避直後の星の重力を一時無視するロジックを導入。
- [x] **インパクト・クッションの性能調整**: 反射係数を 1.0 に引き上げ、完全弾性衝突による快適な回避挙動を実現。
- [x] **予測線（教育的UI）の導入**: ゴーストモジュール装備時、ベースモジュール未装備でも「成功の未来」を表示する仕様に変更。プレイヤーに必要装備を気づかせる学習体験を強化。

### バグ修正・調整 (Fixes & Balance)
- [x] **スター・ブレイカーの装備反映と耐久度合算の完全修正**: ロケットに装備したモジュールが船（ship）に展開されない不整合を解消。また、スタック装備時にオブジェクト参照を共有して耐久度が減らない（または即座に枯渇する）不備を、クローン生成による独立管理で修正。
- [x] **アッセンブルパネルの選択ロジック修正**: Chassis/Logic/Modules の各カテゴリにおけるスロット計算を共通化し、上限を超えた選択や、スタックされたアイテムの選択解除不能バグを解消。
- [x] **アーク・エクスパンダーの残像解消**: 航行終了時の `reset()` 処理において、アーク拡大倍率が残留する不具合を解決。
- [x] **レアブースター（マグネティック・パルス、閃光推進剤）の機能復元**: 発射時に使用したブースターの効果が航行中に正しく反映されるよう、プロパティ参照と初期化ロジックを修正。
- [x] **アイテム命名規則と説明文の標準化**: 「機能名・ゴースト」の形式に統一。また、「直前に破壊（ブレイカー）」「激突時にバウンド（クッション）」と書き分けることで、システム上の優先順位を明確化。
- [x] **Trading Post での耐久力表示の正常化**: 交易所（Trading Post）内のアイテムカードや売却リスト、および解体結果リストにおいて、スタック数バッジや耐久力ゲージが正しく表示されるように改善。
