# Gravity Freight 開発バックログ

完了した項目は [backlog_archive.md](./backlog_archive.md) に移行します。

## Version 2.0.0 に向けたロードマップ

### メタゲーム (Meta-game)
- [ ] リプレイ画面のUI改善: 現在の暫定的なUIをリファクタリングし、より直感的で没入感のある操作体験を実現。

### アーキテクチャ刷新 (Architecture Refactoring)
- [ ] Universe (World) オブジェクトの導入: Game クラスから物理オブジェクト（bodies, ship 等）を分離し、セッション単位でライフサイクルを管理。
- [ ] セッション管理の徹底: タイトル・アーカイブ画面では宇宙を破棄し、プレイ開始時のみ動的に生成する設計へ変更。
- [ ] 描画システムの疎結合化: 宇宙オブジェクトの有無に基づいて描画を自動的に ON/OFF し、null 参照エラーを構造的に排除。
- [ ] タイトル画面用背景の独立: 物理演算用 bodies に依存しない、静的またはプロシージャルなタイトル背景システムを実装。

## Version 3.0.0 に向けたロードマップ

### UI・演出の改善 (UX/UI)
- [ ] アイテムごとの画像設定: カード背景に各アイテムの個別画像を表示し、視認性と没入感を向上。
- [ ] Unit画像の自動生成: 選択された Chassis、Logic、Module を合成し、建造されたロケット固有の画像を自動生成してカード背景に表示。
- [ ] Unitに応じたロケット描画: 現在の汎用形状（青い三角）ではなく、選択された Unit の素材や形状を反映したリッチなロケットモデルが飛行するように拡張。
- [ ] モジュール発動時の視覚効果（FX）の刷新: 「スター・ブレイカー」の破壊粒子、クッションの衝撃波、スラスターの火花など、主要モジュールの発動エフェクトを一貫性のあるプレミアムな品質で実装。
