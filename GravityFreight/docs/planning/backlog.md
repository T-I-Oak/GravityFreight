## Version 1.0.0 に向けたロードマップ

### ゲームシステム (Game System)
- [ ] 「ラスボス」のような大型ターゲットを追加
    - 今のルールだけでは同じステージの繰り返しになってしまうので、5セクターごとなどに何か目標になるものを追加したい
    - 例: star itemsとして回収必須のアイテムを複数個所に追加し、すべて回収するまでアークがdisabledになる（lost扱い）
- [ ] [実験] 重力がマイナスの星を作ったらどうなるか
    - 1個だったらどうなるか
    - すべてだったらどうなるか
    - 新しい体験になりそうなら、「ラスボス」の代わりに追加をしたい
- [ ] ストーリーの導入
    - cargoボーナスを獲得したときにストーリーが進む仕組みを導入する
    - ストーリーはゲームごとに最初の分岐から始まる
    - 出口（3種類）に応じてストーリーが分岐する
        - 1回目で3分岐、2回目で9分岐、3回目で27分岐（ストーリーはここまで）
    - deliveredの報酬アイテムとして仮想的に「手紙」を追加する（フラグを立てるだけ）
    - ゲーム中のスコアー表示のあたりにメールアイコンを追加し、ホバー（タップ）したらメールが読めるようにする
    - メールアイコンは出口に応じたカラーで表示される
    - 未読のメールには未読マークを付ける
    - local storageに、どのメールを取得したかの記録を残す
        - 1回読んだメールは未読マークを付けない

### アイテム (Items)
- [ ] **アイテムカードの説明文の見直し**: 現在のゲームシステムや表現に合わせて、各アイテムの説明文（フレーバーテキスト）の内容を全面的に再確認・修正。
- [ ] スター・ブレイカーで星を破壊したときの動作を修正
    - 星を破壊する直前に重力の影響をかなり受けているのでスピードがかなり早くなっているが、その状態で星が消えると高速で画面外に飛び出してしまうため、速度を抑えるような処理を追加したい

- [ ] 航行中に後ろから追いかけてくる荷物の間隔をもっと短くする

### バグ修正・調整中 (Fixes & Adjustments)
- [ ] **[Bug] アーク・エクスパンダーの残像修正**: 使用後、次のエイム開始時に拡大効果が残ったように表示される問題を修正。
- [ ] **[Bug] スター・ブレイカーの挙動調査**: 搭載しているにも関わらず、星との衝突時に破壊が発動せず墜落（crash）するケースを調査・修正。
- [ ] **Trading Post での耐久度表示**: 施設内のアイテムカードに耐久度ゲージが表示されていない問題を修正。
- [ ] **アイテムカードのホバーステートの分離**: 「非選択&ホバー」と「選択済み&ホバー」を視覚的に明確に区別できるように調整。
- [ ] **マップ閲覧 (View Map) 時の星情報表示**: リザルト画面からマップを閲覧している際、星のホバー情報（Star Item）が確認できない問題を修正。
- [ ] **施設の売買ボタンの視認性向上**: ボタンが disabled の時、背景と区別がつくように枠線などを追加。
- [ ] **ビルドパネルの選択順序緩和**: Rocket や Launcher を選ぶ前でも、Booster を先に選択できるように変更。

### アイテム・データ修正 (Items & Data)
- [ ] **アイテム説明文とパラメタの更新**:
    - **高反応燃料**: 説明文を「発射台の燃料の代わりに使用できる1回分の強化燃料。」に変更し、耐久値を 1 に設定。
    - **高反応燃料パック**: 説明文を「発射台の燃料の代わりに使用できる2回分の強化燃料。」に変更。

## Version 3.0.0 に向けたロードマップ

### UI・演出の改善 (UX/UI)
- [ ] **アイテムごとの画像設定**: カード背景に各アイテムの個別画像を表示し、視認性と没入感を向上。
- [ ] **Unit画像の自動生成**: 選択された Chassis、Logic、Module を合成し、建造された機体固有の画像を自動生成してカード背景に表示。
- [ ] **Unitに応じたロケット描画**: 現在の汎用形状（青い三角）ではなく、選択された Unit の素材や形状を反映したリッチなロケットモデルが飛行するように拡張。
