## Version 1.0.0 に向けたロードマップ

### ゲームシステム (Game System)
- [ ] 「ラスボス」のような大型ターゲットを追加
    - 今のルールだけでは同じステージの繰り返しになってしまうので、5セクターごとなどに何か目標になるものを追加したい
    - 例: star itemsとして回収必須のアイテムを複数個所に追加し、すべて回収するまでアークがdisabledになる（lost扱い）
- [ ] [実験] 重力がマイナスの星を作ったらどうなるか
    - 1個だったらどうなるか
    - すべてだったらどうなるか
    - 新しい体験になりそうなら、「ラスボス」の代わりに追加をしたい

### アイテム (Items)
- [ ] **アイテムカードの説明文の見直し**: 現在のゲームシステムや表現に合わせて、各アイテムの説明文（フレーバーテキスト）の内容を全面的に再確認・修正。

### 航行システム（Flight System）
- [ ] ロケットから射出される波紋のサイズが荷物の回収範囲と一致しているか確認（一致しているなら、回収範囲が広すぎるかも）

### UI・演出の改善 (UX/UI)
- [ ] フライト終了時の演出改善
    - フライトが終了したときに突然飛行処理を止めずに、軌跡が消えるまで処理を続ける
- [ ] **星の破壊エフェクト**: 「Star Breaker」による星の破壊時、粒子が飛び散るような視覚効果を追加。
- [ ] **背景のワープ演出**: ステージクリア（セクター移動）時、背景の星が加速し、現在のステージが手前に拡大消去、次のステージが奥から拡大出現する演出。
- [ ] **状況に応じた動的エフェクト**: 加速時やアイテム取得時、および境界付近での警告演出などの強化。
- [ ] 飛行終了時のエフェクト（crash/goal/lost/return）
- [ ] **ストーリーダイアログのアイコンスタイル統一**: ストーリーのダイアログで使用するアイコンを、施設アイコンと同一のスタイル（重厚なマットデザイン）に合わせる。
- [ ] **施設アイコンの文字位置調整**: 施設内のテキストアイコンで、文字が中央に配置されるように調整。
- [ ] **ゲームオーバー画面のスライド演出復活**: ゲームオーバー時のレシート表示において、下からスライドインするアニメーション（印刷プロセスを想起させるもの）を再度適用。
- [ ] SEの追加

    - [ ] ロケットから出るソナー音
    - [ ] ロケットの速度に応じたトーン
    - [ ] アイテム取得音
    - [ ] ロケットの到着位置に応じた音 (crash / goal / lost / return)
    - [ ] UIの操作音

## Version 2.0.0 に向けたロードマップ

### メタゲーム (Meta-game)
- [ ] **How to Play の更なるグラフィカル化・動的化**: アニメーションやデモ映像を用いた、より直感的な操作説明へのアップグレード。
- [ ] **ベストショット・リプレイ**: 1回の発射で最高スコアを記録した際のプレイを再現。
    - 発射時の初期状態（重量、パワー、角度、アイテム効果等）を記録し、物理演算を再実行。
- [ ] アチーブメントシステムの追加
    - 以下のようなアチーブメントを管理・表示できるようにする
        - XX回発射（XXは3段階くらい。以下同様）
        - XX機失う
        - セクターXX到達
        - XX回配達（deliveredを達成）
        - 1回の航行距離XXを達成
        - 累積航行距離XXを達成
        - 最高点XX点を達成
        - 累積XX点を達成
        - 最高所持コインXXを達成
        - 累積でXXコイン稼ぐ
        - 累積でXXコイン消費
        - Trading Post / Repair Dock / Black Market にXX回到達
        - XX回アイテムを購入
        - XX回アイテムを売却
        - XX回アイテムを強化
        - ストーリーを見る（T分岐、R分岐、B分岐、すべて）

## Version 3.0.0 に向けたロードマップ

### UI・演出の改善 (UX/UI)
- [ ] **アイテムごとの画像設定**: カード背景に各アイテムの個別画像を表示し、視認性と没入感を向上。
- [ ] **Unit画像の自動生成**: 選択された Chassis、Logic、Module を合成し、建造された機体固有の画像を自動生成してカード背景に表示。
- [ ] **Unitに応じたロケット描画**: 現在の汎用形状（青い三角）ではなく、選択された Unit の素材や形状を反映したリッチなロケットモデルが飛行するように拡張。
