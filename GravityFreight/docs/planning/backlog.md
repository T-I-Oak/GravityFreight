# Gravity Freight 開発バックログ

完了した項目は [backlog_archive.md](./backlog_archive.md) に移行します。

## Version 1.0.0 に向けたロードマップ

### アイテム (Items)
- [ ] **アイテムカードの説明文の見直し**: 現在のゲームシステムや表現に合わせて、各アイテムの説明文（フレーバーテキスト）の内容を全面的に再確認・修正。

### UI・演出の改善 (UX/UI)
- [ ] **ビルドパネルのカテゴリー余白調整**: カテゴリー名とアイテム間の余白を詰め、上のカテゴリーとの間隔を広げる（全体の高さは変えずにバランスを調整）。
- [ ] **用語定義の統一**: 「機体」という言葉が「ロケット全体」と「シャーシ」の2本立てで使われているため、ゲーム全体の見直しを実施し、混同を解消する。
- [ ] **施設名称とイニシャルの変換共通化**: Trading Post <-> T などの変換が各所に散在しているため、Data.js 等に集約して共通化する。

### バグ修正・調整中 (Fixes & Adjustments)
- [ ] **高セクターでの脱出難易度上昇**: セクターが進むごとに脱出が難しくなる仕組みを追加する。
    - 母星の重力を大きくする、星全体の重力を大きくする、発射パワーが減っていくあたりを試して調整する。
- [ ] **インパクト・クッションの母星反発バグ**: インパクト・クッション搭載機が母星に帰還しようとすると反発してしまい、帰還できない問題を修正。
- [ ] **ズーム率の永続化**: マップのズーム率を localStorage に保存し、次回起動時やリロード時にも前回のズーム設定を維持するようにする。


## Version 2.0.0 に向けたロードマップ

### メタゲーム (Meta-game)
- [ ] **星の破壊エフェクト**: 「Star Breaker」による星の破壊時、粒子が飛び散るような視覚効果を追加.
- [ ] **ベストショット・リプレイ**: 1回の発射で最高スコアを記録した際のプレイを再現。
    - 発射時の初期状態（重量、パワー、角度、アイテム効果等）を記録し、物理演算を再実行。
- [ ] アチーブメントシステムの追加
    - 以下のようなアチーブメントを管理・表示できるようにする
        - XX回発射（XXは3段階くらい。以下同様）
        - XX機失う
        - セクターXX到達
        - XX回配達（deliveredを達成）
        - 1回の航行距離XXを達成
        - 累積航行距離XXを達成
        - 最高点XX点を達成
        - 累積XX点を達成
        - 最高所持コインXXを達成
        - 累積でXXコイン稼ぐ
        - 累積でXXコイン消費
        - Trading Post / Repair Dock / Black Market にXX回到達
        - XX回アイテムを購入
        - XX回アイテムを売却
        - XX回アイテムを強化
        - ストーリーを見る（T分岐、R分岐、B分岐、すべて）

## Version 3.0.0 に向けたロードマップ

### UI・演出の改善 (UX/UI)
- [ ] **アイテムごとの画像設定**: カード背景に各アイテムの個別画像を表示し、視認性と没入感を向上。
- [ ] **Unit画像の自動生成**: 選択された Chassis、Logic、Module を合成し、建造された機体固有の画像を自動生成してカード背景に表示。
- [ ] **Unitに応じたロケット描画**: 現在の汎用形状（青い三角）ではなく、選択された Unit の素材や形状を反映したリッチなロケットモデルが飛行するように拡張。
