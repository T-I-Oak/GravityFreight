# 演出および物理・音響同期仕様 (Presentation & Physics Sync)

本ドキュメントは、アニメーション演出時の物理計算の停止・再開、および描画同期とオーディオ再生に関する技術的なメタ制約を定義します。

## 1. アニメーションと描画フレームの同期

CSS トランジションや動的な描画リセットを JavaScript からトリガーする際に、画面のチラつき（表示不整合）を防ぐための原則です。

### 1.1 `requestAnimationFrame` (rAF) の活用
- 非表示状態（`.hidden`等）から復帰させた直後に CSS アニメーションを開始させる場合、ブラウザの描画キューが確定するのを待つ必要がある。
- よって、UIクラス操作の直後に直接アニメーションクラスを付与するのではなく、必要に応じて rAF で次の描画フレームを待ってから適用すること。

### 1.2 背景描画のループバック保護
- 動的スターフィールド等において、Z軸座標がリセット（画面手前から奥へループ）されるフレームでは、流線（Streaks）の描画処理を抑制すること。
- これを怠ると、画面全体を横切る巨大な光のノイズ（フラッシュ）が発生する。

## 2. 物理演算と演出フェーズの分離 (Finishing Phase)

航行の成功・失敗の判定直後（演出フェーズ）におけるデータ確定の掟です。

- **確定と停止**: 判定の瞬間、その座標や取得アイテムなどの情報をリザルト用に格納し、以後の追加計算（新たな物理移動や当たり判定）を完全に停止させる。
- **物理的終息の表現**: 物理ループ自体は止めず、ロケットの座標更新のみを止めることで、末尾の軌跡（トレイル）が自然に一点へ収束する処理のみを継続させる。人為的なアルファチャンネル（透過度）操作による唐突な消滅は行わない。

## 3. オーディオ発音制限 (Audio System Constraints)

音響制御において、音割れやCPU負荷によるノイズを防ぎ、プレイ体験を守るための制約です。

- **キューの待機と遅延**: ワープ演出やセクター生成など、負荷の高い処理と同時に音を鳴らすと音響が乱れる場合がある。この際、発音（または重い処理）を `setTimeout` 等で50ms程度遅延させ、発音処理とメインスレッドのピーク負荷を分散させること。
- **標準的な減衰（Envelope）**: 短いUI操作音であっても、音の終端でのプツッというノイズ（クリッピング）を防ぐため、`exponentialRampToValueAtTime` を用いて最低でも50msかけてフェードアウトする減衰曲線を標準とすること。
- **マスターボリューム制御の時定数**: ボリューム変更は即座に反映するのではなく、時定数をもたせて滑らかに反映させる。
