# ターミナル・レポート（ゲームオーバー画面）デザイン仕様書

## 1. 概要
ターミナル・レポートは、高精細な「感熱紙のレシート」の質感を再現したデザインです。ギザギザの切り取り線、等幅フォント、および表示時にアニメーションする「公印（スタンプ）」が視覚的な特徴です。

## 2. レイアウトとタイポグラフィ
- **幅**: 最大 460px。レスポンシブ対応。
- **フォント**: Courier New（レシート本文）、Inter（公印）。
- **インク色**: 
  - メイン: `#1a1a1a`（黒）
  - アクセント（本文中のグレード / ランク上位3位）: `#b22d2d`（赤インク）
- **フッターのバランス**: バーコードの下に 5px の空白を残し、公式文書らしい密度を維持します。

## 3. 公印（スタンプ）の構成
スタンプは、2つの長方形を組み合わせた構造です：左側の枠（テキスト詳細）と右側の枠（グレード記号）。

### 3.1 比率
「SSグレード（Hugeサイズ）」をベースラインの「美的基準」として設計されています。
- **基準幅**: コンポーネントの内容（水平方向の余白とパディング）によって決定。
- **内部比率**: 左フレーム（約71%） : 右フレーム（約29%）。
- **グレード記号（右側）**: 常に垂直・水平方向に中央揃え。

### 3.2 スケーリング論理（数学的近似）
全グレードで一貫した「見た目」を維持するため、すべてのプロパティは Huge バリアントを基準にスケールします：
- **Huge (SS)**: 100% (基準)
- **Normal (S - D)**: 約 78% スケール
- **Mini (E)**: 約 70% スケール

### 3.3 黄金律 (Huge基準)
- **文字間隔 (GRADE)**: `8px`（スケールダウン時に個別に縮小しないこと）。
- **左パディング**: `15px`。
- **右パディング**: `20px`。
- **境界線の角丸**: `0px`（結合された長方形としての鋭角な角）。

## 4. インタラクションとアニメーション
- **用紙のスライドイン**: 0.8s, cubic-bezier(0.2, 1, 0.3, 1)。
- **スタンプの押印**: 0.4s。ランダムな回転（-10度〜-25度）と、わずかな位置オフセットを伴うインパクトアニメーション。
- **終了アクション**: 「END CONTRACT」をクリックすると、ゲーム状態をリセットする前に 0.8s のスライドダウン演出を実行。

## 5. CSS クラス
- `.report-official-seal`: コンテナ。
- `.huge`, `.normal`, `.mini`: スケールバリアント。
- `.receipt-grade-ss`, `.receipt-grade-s`: 詳細行のカラーオーバーライド（赤インク）。
- `--stamp-rot`, `--stamp-x`, `--stamp-y`: JS 経由で注入される動的なランダム変数。
