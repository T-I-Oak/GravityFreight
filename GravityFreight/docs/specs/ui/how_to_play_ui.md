# How to Play UI 仕様書 (TutorialUI)

## 概要
ゲームの基本操作を説明するためのオーバーレイUI。
スライド形式で情報を表示し、静的なテキスト、アイテムカード、および動的なキャンバスアニメーションを組み合わせて提示する。

## コンポーネント構成
- **TutorialUI.js**: メインコントローラー。UIの生成、ページ遷移、アニメーションの同期を管理する。
- **TutorialSlidesData.js**: 各スライドの内容を定義するデータ。
- **TutorialDiagrams.js**: スライド内のキャンバス描画とシミュレーションアニメーションを担当する。

## 主要機能
### 1. データ駆動型レンダリング
`slidesData` に定義されたオブジェクトに基づき、HTML構造を自動生成する。
- `layout`: 'full' または 'split' (2カラム) を選択可能。
- `blocks`: 'info' または 'tip' タイプの情報ブロック。
- `cards`: `ITEM_REGISTRY` から参照されるリッチなアイテムカードの表示。
- `canvasId`: 動的な図解を表示するためのターゲットID。

### 2. 非同期ページ遷移演出
背景画像を際立たせるため、以下のシーケンスで遷移を行う。
1. **フェードアウト**: 現在のコンテンツを `opacity: 0` に変化（0.4s）。
2. **スライド**: 背景（トラック）を左右に移動（0.6s）。
3. **フェードイン**: 
   - タイトル（h3）を表示（delay 0.1s）。
   - 本文（info-block等）を `translateY` を伴って表示（delay 0.3s）。

### 3. グロスモーフィズム・デザイン
- 背景透過度を極限まで高めたデザイン（`background: none`）。
- `backdrop-filter: blur(6px)` による動的なぼかし効果。
- ブラーは `0px` から `6px` へと数値的にアニメーションし、パッとかかる現象を防止。

### 4. 視認性向上のための工夫
- **テキスト縁取り (.text-outline)**: 背景画像が明るい場合でも可読性を確保するため、1px の黒い縁取りを適用。
- **スクロールバーの動的制御**: アニメーション中（`is-animating` クラス）は、全スクロールバーを一時的に非表示にする。

## 5. シミュレーション図解
以下のスライドで動的なデモを実行する。
- **ASSEMBLE**: カード選択とボタンホバーの擬似操作。
- **LAUNCH**: ロケットの発射角度調整と予測線のデモ。
- **NAVIGATION**: 重力スイングバイの物理シミュレーション。

## 用語定義に関する注意
- 現在「機体」という言葉が「ロケット全体」と「シャーシ」の混同した状態で使われている。
- 今後の全体系の見直し（バックログ）において、これらを「ロケット」「シャーシ」等に厳密に分離する予定。
