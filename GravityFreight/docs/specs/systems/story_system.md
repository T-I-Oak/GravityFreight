# ストーリーシステム（メール機能）仕様書

## 1. 概要
「GRAVITY FREIGHT」の世界観やキャラクターの背景を断片的に提示し、プレイヤーに探索の動機付けを与えるシステムです。特定の貨物を目的地に届けることで「メール（通信）」が解放され、それを読み進めることで物語が進行します。

## 2. データ構造
ストーリーは、配送先（出口）に対応した3つの系列（ブランチ）で構成されます。

- **系列 (Branch)**:
    - `T` (Trading Post): 商人や家族に関連する物語
    - `R` (Repair Dock): 整備士やロケットに関連する物語
    - `B` (Black Market): AIや軍事・実験に関連する物語
- **要素 (Step)**:
    - 各系列は 1 〜 3 のステップで進行します（例: T1 -> T2 -> T3）。
    - 配送が成功するたびに、その系列の次のステップが解放されます。
- **データ項目**:
    - `id`: 文字列 (例: "T1", "TR2", "TTT3" など。 branching logic に応じて命名)
    - `title`: ストーリーのタイトル
    - `discovery`: 発見時の短いメッセージ（「〇〇に隠されていた手紙」など）
    - `content`: 本文
    - `type`: 系列 ('T', 'R', 'B')

## 3. 解放ロジック
### 3.1 解放のトリガー
- `MissionSystem.resolveItems` 内で、貨物の「一致配送ボーナス」が確定した瞬間に判定を行います。
- 届けた出口の ID に応じて、対応する系列のストーリーを解放します。

### 3.2 進行ルール
- 初回配送時: 系列の Step 1 (T1, R1, B1) を解放。
- 以降の配送時: 
    - 現在の到達 Step に基づき、次の Step を解放。
    - ※将来的に分岐（T -> TT or TR）を実装する場合、前回のゴール履歴やセクター数を参照しますが、v0.15 では `local_story.txt` に準拠した直線、または単純な分岐を実装します。

## 4. UI仕様
### 4.1 メールアイコン (Mail Button)
- **表示場所**: メインHUDの右側、またはスコア表示の近く。
- **状態**:
    - **通常**: アイコンが表示されているが特に変化なし。
    - **未読あり**: 新しいストーリーが解放されたがまだ読まれていない場合、アイコンが「呼吸（Pulsing）」するように明滅します。
- **カラーコード**:
    - アイコンの色は `Data.CATEGORY_COLORS.CARGO` (アクア) を基本とし、最後に解放された系列の色 (`GOAL_COLORS`) がアクセントとして使われます。

### 4.2 ストーリーモーダル
- アイコンクリックで展開。
- カード形式のデザインを採用し、以下の情報を表示:
    - 分岐アイコン (T/R/B)
    - タイトル
    - 発見場所・状況 (Discovery Message)
    - 本文 (Main Content)
- 「閉じる」ボタンでゲーム画面に戻ります。

## 5. 永続化と管理 (StorySystem)
- **内部ステート**: 解放済みIDの配列、既読IDの配列、現在の系列ごとの進捗。
- **保存先**: `localStorage` (Key: `gravity_freight_story_progress`)。
- **リセット**: `fullReset` 時にはストーリー進捗はリセット**されません**（コレクション要素としての性質を持つため）。
