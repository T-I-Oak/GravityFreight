## Version 1.0.0 に向けたロードマップ

### メタゲーム (Meta-game)
- [ ] **ランキング機能の実装**:
    - **Sector ランキング**: 到達した最大セクターの記録。
    - **Score ランキング**: 累計スコアの記録。
    - **Delivery ランキング**: 累計貨物配送数の記録。
- [ ] **ベストショット・リプレイ**: 1回の発射で最高スコアを記録した際のプレイを再現。
    - 発射時の初期状態（重量、パワー、角度、アイテム効果等）を記録し、物理演算を再実行。

### ゲームシステム (Game System)
- [ ] ロケットから射出される波紋のサイズが荷物の回収範囲と一致しているか確認（一致しているなら、回収範囲が広すぎるかも）
- [ ] 「ラスボス」のような大型ターゲットを追加
    - 今のルールだけでは同じステージの繰り返しになってしまうので、5セクターごとなどに何か目標になるものを追加したい
- [ ] [BUG] リザルト画面で goal bonus が以前のセクターからの累積になっている

### アイテム (Items)
- [ ] **アイテムカードの説明文の見直し**: 現在のゲームシステムや表現に合わせて、各アイテムの説明文（フレーバーテキスト）の内容を全面的に再確認・修正。
- [x] ロスト保険のバランス調整: Lostしたときだけでなく、Crashしたときにも保険金が受け取れるように変更。
- [x] アイテム名変更
    - "標準発射台 [LNC-N]" → "標準発射台 [LN1200-N]" (N は 2～4)
    - "精密発射台 [PRC-N]" → "精密発射台 [PR1000-N]" (N は 2～4)
- [ ] スター・ブレイカーで星を破壊したときの動作を修正
    - 星を破壊する直前に重力の影響をかなり受けているのでスピードがかなり早くなっているが、その状態で星が消えると高速で画面外に飛び出してしまうため、速度を抑えるような処理を追加したい
- [x] ゴール・エクスパンダー → アーク・エクスパンダー に名称変更
    - アークサイズ 1.2倍 → 2.0倍 に変更

### UI・演出の改善 (UX/UI)
- [ ] **星 host アイテム表示の最適化**: 1つの星に大量のアイテム（Cargo/Parts）がある場合、ホバー表示が画面外にはみ出す問題の解消。
    - アイテムリストのグリッド表示化、または「...他○件」といったサマリー表示の検討。
- [ ] **ゲームオーバー画面機能追加**: ランキング表示の追加、総合評価（グレード）の表示追加
- [ ] 初心者にもっと分かりやすいボタンへの変更
    - ボタンへのサブテキスト追加
        - "ASSEMBLE ROCKET" → サブテキストに "シャーシとロジックを選択して建造"
        - "LAUNCH ENGINE" → サブテキストに "ロケットと発射台を選択して発射"
    - サブテキストのアニメーション（待機中のロケットなしのサブテキストと同様のアニメーション）
        - ボタンがdisabledの間だけアニメーションするようにする
- [ ] フライト終了時の演出改善
    - フライトが終了したときに突然飛行処理を止めずに、軌跡が消えるまで処理を続ける
- [ ] ゲームオーバー画面で "END" を押したとき、レシートが画面下に下がって消えるようにする（その後タイトル画面に遷移する）
- [ ] タイトル画面のロケット軌道（角度）の調整
- [ ] **星の破壊エフェクト**: 「Star Breaker」による星の破壊時、粒子が飛び散るような視覚効果を追加。
- [ ] **背景のワープ演出**: ステージクリア（セクター移動）時、背景の星が加速し、現在のステージが手前に拡大消去、次のステージが奥から拡大出現する演出。
- [ ] **状況に応じた動的エフェクト**: 加速時やアイテム取得時、および境界付近での警告演出などの強化。
- [ ] 飛行終了時のエフェクト（crash/goal/lost/return）
- [ ] SEの追加

## Version 3.0.0 に向けたロードマップ

### UI・演出の改善 (UX/UI)
- [ ] **アイテムごとの画像設定**: カード背景に各アイテムの個別画像を表示し、視認性と没入感を向上。
- [ ] **Unit画像の自動生成**: 選択された Chassis、Logic、Module を合成し、建造された機体固有の画像を自動生成してカード背景に表示。
- [ ] **Unitに応じたロケット描画**: 現在の汎用形状（青い三角）ではなく、選択された Unit の素材や形状を反映したリッチなロケットモデルが飛行するように拡張。
