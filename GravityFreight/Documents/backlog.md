## 1. アイテム抽選システム (Game.js) - [継続 TODO]

### [ISSUE] 抽選アルゴリズムの不整合
現在の実装は固定重み方式（COMMON=100, UNCOMMON=30, RARE=10）ですが、最新の仕様では「しきい値と出現率の差分を重みとする逆転方式」が採用されました。

### [TODO]
- [ ] **`getSectorItemThreshold()` の修正**
    - 現状: `9 + (this.stageLevel || 1)` (第1ステージで 10)
    - 目標: `14 + (this.stageLevel || 1)` に変更 (第1ステージで 15 となるように調整)。
- [ ] **`getWeightedRandomItem()` の修正**
    - 現状: 固定重み分岐（100, 30, 10）
    - 目標: 重み計算を `const weight = Math.max(0, threshold - item.rarity)` に変更。
- [ ] **アイテムの除外ロジックの強化**
    - 重みが 0 以下の場合は、単に 0 とするのではなく、抽選リスト（`items` 配列）自体から除外すること。
- [ ] **thresholdBonus の反映確認**
    - ブラックマーケット等の `thresholdBonus` 加算が、上記の新重み計算に正しく反映されることを再検証。

## 2. 拠点イベント (Goal Events) - [実装済み確認完了]

以下の項目は、コード精査により現在の `Game.js` で既に仕様通り実装されていることを確認しました。

### [x] 交易所 (Trading Post)
- [x] **在庫生成**: パーツのみ、ランダムに **6 つ** を選出。
- [x] **販売価格**: 各アイテムの査定価格の **2 倍** を基準価格として算出（セール品はそこから割引）。
- [x] **対象除外**: 抽選から CARGO および COIN が正しく除外されていることを確認。

### [x] 整備ドック (Repair Dock)
- [x] **修理対象の拡張**: 現在の装備品だけでなく、**インベントリ内の全発射台** が修理候補に並ぶ。
- [x] **強化項目の分離**: 「予測性能向上 (precision)」と「回収性能向上 (pickup)」が独立した抽選項目として分離されている。
- [x] **強化時の例外処理**:
    - [x] `Gravity Multiplier` が 0.1 以下の場合は重力安定化強化をスキップ（オプションから除外）。
    - [x] 耐久度プロパティ（`maxCharges`）を持たないアイテムは耐久性向上をスキップ（オプションから除外）。

### [x] 闇市場 (Black Market)
- [x] **ループ抽選ロジック**: 
    - 100c コース：合計査定額が **100c** 以上になるまで、獲得アイテムの追加抽選を繰り返す。
    - 500c コース：合計査定額が **600c** 以上になるまで、追加抽選を繰り返す。
