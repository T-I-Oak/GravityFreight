# Gravity Freight 開発バックログ (v1.0.0に向けたロードマップ)

初版リリースに向けて必要な機能をカテゴリ別に整理しました。

## 0. 重要バグ修正 (Critical Bug Fixes)
- [x] **衝突判定の突き抜けバグの修正 (v0.4.1)**: 高速移動時に星を通り抜けてしまう現象（物理ステップを飛び越える、またはCCDの判定漏れ）の再検証とCCD（連続衝突判定）の強化。
- [x] **アセンブリ時のスロット数超過バグ (v0.4.1)**: モジュール選択後にスロット数の少ないシャーシへ変更した際、制限を超えて搭載できてしまう問題の修正。
- [x] **ブースターの不正消費バグ (v0.4.2)**: 燃料を選択した状態で機体を建造すると、発射前にもかかわらず燃料が消費されてしまう問題の修正。
- [x] **【不具合疑い】精密発射台のパラメータ (v0.4.3)**: 「予測線強化」と記載されているが、現在のパラメータ設定が意図した通りの効果を発揮しているか再検証。
- [x] **ロスト保険の二重計上と内訳非表示 (v0.4.14)**: 保険金が獲得コインと重複し、内訳に出ない問題を修正。
- [x] **星のアイテムカウントバッジ (v0.4.14)**: 重複アイテムの数量表示 [x 2] が表示されない問題を修正。
- [ ] **不一致の荷物を持ってゴールしたとき、リザルト画面で「Cargo (unmatched)」と「Cargo Reword」で重複した値が表示される**
- [ ] **発射準備時の向きの不一致**: ドラッグでのエイム時、船の座標は更新されるが向き（rotation）が同期されず、発射時に意図しない向きになる。（※再現方法が判明した時点で着手）

## 1. 新仕様に伴う追加修正タスク (Spec-v1.0.0 Updates)

### 1.1 アイテム抽選システム (Game.js) - [継続 TODO]
仕様書 (`spec.md`) 第7章に基づき、抽選ロジックを刷新する必要があります。
- [ ] **`getSectorItemThreshold()` の修正**
    - 現状: `9 + stageLevel` (10から開始)
    - 目標: `14 + stageLevel` (15から開始) に変更。
- [ ] **`getWeightedRandomItem()` の修正**
    - 現状: 固定重み分岐（100, 30, 10）
    - 目標: 重み計算を `const weight = Math.max(0, threshold - item.rarity)` に変更。
- [ ] **thresholdBonus の反映**: Black Market 等のボーナスが、新重み計算に正しく反映されることを確認。

- [x] **闇市場 (Black Market)**: 査定合計額（100c/600c）に達するまでのループ抽選。
