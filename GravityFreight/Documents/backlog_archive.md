## Version 0.7.0 (2026/04/02) - 仕様整合性と経済システムの安定化

### 仕様とテストの整合性確保 (Specs & Tests)
- [x] **目的地報酬および配送ボーナスの数値検証とロジック修正**: `07_collection_economy.test.js` を作成し、基礎報酬 (+2000~5000) とボーナス (+1500) の合算を検証・修正。
- [x] **出現率とセクターしきい値の重み付け抽選の検証**: `14 + S` の計算式とレアリティごとの重み付けが仕様通りであることをテストで証明。
- [x] **複数保険モジュールの重複支払い検証**: 保険モジュールの装備数に応じた保険金支払いロジックの正当性を検証。

### 貨物・経済・バランス調整 (Economy & Balance)
- [x] **デバッグコードの削除と配置数の正常化**: `MissionSystem.js` 内のアイテム強制注入コードを削除し、星ごとのアイテム配置数（1〜2個）の制約を遵守。
- [x] **ゴール通過報酬の正常化**: 貨物を所持していない場合でも、ゴール到達時にボーナスアイテムが付与されるように改善。
- [x] **アイテムデータの日本語化と仕様書整合**: `Data.js` の貨物アイテム名を日本語に統一し、`spec.md` の説明文と完全一致するように修正。

## Version 0.9.0 (2026/04/02) - アイテムバランス刷新と保険システムの拡張

### アイテム・バランス調整 (Items & Balance)
- [x] **発射台およびブースターの名称刷新と性能調整**:
    - 発射台を `LN1200-N` / `PR1000-N` 系列へ型番化し、直感的な性能把握を可能に。
    - 「アーク・エクスパンダー」への名称変更と、拡大倍率の強化（1.2倍 → 2.0倍）。
- [x] **ロスト保険の適用範囲拡大**:
    - `LOST` だけでなく `CRASHED` 時にも保険を適用。
    - 貨物を所持していない機体消失時でもパーツ査定額を支払うように修正。

### バグ修正と品質向上 (Bug Fixes & Quality)
- [x] **リザルト画面のゴールボーナス累積バグの修正**: `Game.reset()` 時のデータ初期化漏れを解消。
- [x] **自動テストによる品質保証**: 経済システムおよびアイテムデータの整合性を検証するテストスイートの拡充。

## Version 0.9.1 (2026/04/03) - UIのアクセシビリティ向上と視認性のブラッシュアップ

### UI・演出の改善 (UX/UI)
- [x] **初心者にもっと分かりやすいボタンへの変更**:
    - ボタンへのサブテキスト追加（"ASSEMBLE ROCKET" / "LAUNCH ENGINE"）。
    - 建造: 「シャーシとロジックを選択すると建造できます」
    - 発射: 「ロケットと発射台を選択すると発射できます」
    - ボタンが `disabled` の間のみサブテキストを水色パルス明滅させるアニメーションを実装。
- [x] **UIの視認性と配色の改善 (UI Polish)**:
    - `disabled` 時の `opacity` 指定を廃止し、背景と彩度の調整で状態を表現することで、テキストのくすみを解消。
    - サブテキストの配色をブランドカラー（水色）とグレーの動的切り替えに調整。
- [x] **レイアウトの最適化**:
    - `#launch-control` パネルの幅を 320px に拡張し、文言の長さに対応。
    - モバイル環境（レスポミシブ）でのフォントサイズと文字間隔の自動調整を実装。

## Version 0.9.2 (2026/04/03) - UX 高速化と開始演出の導入

### UI・演出の改善 (UX/UI)
- [x] **起動パフォーマンスの最適化 (Lag Elimination)**: 
    - タイトル画面からビルド画面への遷移時に設定されていた 1秒間の固定遅延 (`setTimeout`) を削除。
    - `index.html` の主要パネルに初期 `hidden` クラスを付与し、レンダリング時のチラつきを抑止。
- [x] **セクター開始演出「SECTOR READY」の追加**: 
    - 新しいセクター（ゲーム開始時および拠点終了時）の開始をプレイヤーに伝えるため、画面中央にアニメーション（ぼかし解除・フェードイン・上昇フェードアウト）付きの通知を表示。
- [x] **リザルト画面の表示改善**: 
    - アイテム獲得なし時の文言を「回収アイテムなし」に固定し、点滅アニメーションを削除。
    - 他のパーツスロットと一貫性のある「枠線あり」のプレースホルダースタイルに統一。

### パフォーマンスと品質 (Performance & Quality)
- [x] **UI レンダリングの最適化**: 
    - `UISystem.renderList` に `DocumentFragment` を導入。インベントリリスト描画時の DOM 操作回数を削減し、タブ切り替え時のレスポンスを改善。
- [x] **バージョン管理とキャッシュ対策の徹底**: 
    - `Game.js`, `index.html`, `package.json` のバージョンを `0.9.2` に一括更新。
    - `index.html` の外部スクリプト読み込みに `?v=0.9.2` 形式のクエリパラメータを付与し、キャッシュによる不整合を防止。
- [x] **自動テストによるデグレード防止**: 
    - 既存の全 72 件のテストがパスすることを確認し、バージョン更新に伴うテスト期待値の同期も完了。
